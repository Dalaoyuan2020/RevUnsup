# 阈值陷阱: pow(thresh, 2.0)

## 一句话总结
**封装版的 anomalyDetMatin 函数第一行就 thresh = pow(thresh, 2.0), 你传 0.1 实际生效的是 0.01。文档没写, 不知道这个所有调参都是错的。**

## 来源 (DogWind 在 RU-001 里发现)

`infer/demo.cpp` (decrypted) 函数 `anomalyDetMatin` 的第一行:
```cpp
thresh = pow(thresh, 2.0);
```

## 换算表 (强制贴在脑门上)

| 你传入 | 实际生效 | 备注 |
|---|---|---|
| 0.05 | 0.0025 | 极敏感 |
| 0.10 | 0.01 | main 函数默认 |
| 0.15 | 0.0225 | |
| 0.20 | 0.04 | |
| 0.30 | 0.09 | |
| 0.50 | 0.25 | 较松 |
| 1.00 | 1.00 | 几乎不检出 |

## 反算公式

如果你想要【实际生效】= X, 应该传入 = sqrt(X)
```python
threshold_to_pass = math.sqrt(actual_threshold)
```

## 强制标注规则

任何文档/代码/commit message 提到阈值时, 必须写成:
```
thresh_input=0.1 (thresh_effective=0.01)
```

不允许只写 `thresh=0.1` 这种模糊形式, 否则下一个看的人会被坑。

## 为什么这是一个【陷阱】

文档没写。函数名叫 `anomalyDetMatin`, 谁也不会想到它内部偷偷把传入参数平方。

如果你做调参实验, 用 `thresh = 0.1` 跑一组, 用 `thresh = 0.2` 跑另一组, 你以为的"差 2 倍", 实际是"差 4 倍"。所有的 PR 曲线/F1 计算全错。

## 跟其他陷阱的关系

封装版还有 2 个陷阱 (RU-001 第 8 题):
1. **本节: pow(thresh, 2.0)** ← 最坑
2. **输出是灰度热力图, 不是二值 mask** — 调用方要自己 threshold
3. **隐藏的可调参数** ([[隐藏可调参数]]) — modelInputW/H 等被硬编码

三个陷阱合在一起 = 任何人不读 RU-001 直接调封装版, 第一次实验结论就是错的。

## 已落地的防护

`docs/阈值校准协议.md` (DogWind 在 RU-003 里写的) 已经把这个陷阱作为【强制规则】, 任何涉及阈值的实验都必须先通过协议的 Step A (显式声明 pow 陷阱) + Step D (反平方算回真实传入值)。
