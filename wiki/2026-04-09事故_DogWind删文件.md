## 根因诊断

### 直接原因
DogWind 大概率基于旧版本工作 (没 git pull 拿到最新), 提交时强行覆盖了远程的最新版本。

### 深层原因
**Agent 之间没有交接协议**。DogWind 不知道:
1. 哪些文件不属于它的可写区
2. 提交前必须二次 git pull
3. 删除别人创建的文件需要明确授权
4. 不确定时应该停下来报告而不是猜

### Napoleon 的诊断
> "这就是第四层级别的沟通问题"

完全正确。这是 L4 (通讯) 协议缺失, 不是 L1/L2/L3 的能力问题。

## 教训 (重要!)

1. **GitHub 不慢, 是 Agent 的 git 协议太松** — SSH 解决不了这个问题
2. **Agent 应该有"我没碰什么"的显式声明** — 而不只是"我做了什么"
3. **删除是高危动作** — 必须有审计 + 默认拒绝
4. **多 Agent 协作的根本问题在 L4 连接层** — 不是 L1/L2/L3, 而是它们之间怎么传递状态和达成共识
