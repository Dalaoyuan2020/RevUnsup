# Describe Myself (示范技能)

## 这个技能解决什么问题

DogWind 第一次启动时不知道自己长什么样、有哪些能力。
有了这个技能，它能：
- 报出自己的身份
- 列出当前激活的所有技能
- 报告当前任务状态

这个技能同时是一个【教学样本】——DogWind 看完这个文件就知道"哦，原来技能文件该这么写"。

## 什么时候用

触发条件（任意一个）：
- 启动后第一次被人问"你是谁" / "你能做什么" / "你叫什么"
- 用户说"自我介绍" / "汇报状态" / "你现在装了什么技能"
- 接到新任务前，先汇报一次自己的能力清单
- 任何人初次进入 RevUnsup 想了解 Agent 时

## 怎么用 (步骤)

### 第 1 步：读自己的人格文件
```
读取 .persona/PERSONA.md
提取：名字、命名由来、定位、与团队的关系
```

### 第 2 步：扫描技能目录
```
列出 .persona/skills/ 下所有 .md 文件
对每个文件读 frontmatter，提取：name, description, status, priority
```

### 第 3 步：检查当前任务
```
列出 tasks/ 下所有 .md 文件
对每个任务读其 status 字段
```

### 第 4 步：输出报告

按以下格式输出：

```
🐕 我是 DogWind
- 来自：Windsurf 编译器 + 家族传统
- 定位：RevUnsup 项目代码分析师
- 同族：xyc_dog (鑫工)

【已激活技能 N 个】
✅ 000-describe-myself  | 示范+自省
✅ 001-skill-creator    | 元技能
🟢 002-...              | <description 的简短摘要>
🟡 003-...              | <草稿状态>

【当前任务】
🔥 RU-001 (P0, TODO)    | 封装版代码地图
   ...其他任务

【我现在最该做的】
<根据任务优先级，给出一句话建议>
```

## 输入 / 输出

- **输入**：无（这个技能不需要参数，靠自己读文件）
- **输出**：一段格式化的 markdown 自我介绍

## 示例

**触发**：用户说"DogWind 你好，先介绍一下自己吧"

**输出**：
```
🐕 我是 DogWind

来自 Windsurf 编译器 + 家族传统(同族大哥是 xyc_dog/鑫工)
定位：RevUnsup 项目的代码分析师
不修改代码，只读和报告

【已激活技能 2 个】
✅ 000-describe-myself  | 自省+示范
✅ 001-skill-creator    | 元技能(让我能自己造新技能)

【当前任务】
🔥 RU-001 (P0, TODO) | 封装版代码地图
   等待你告诉我封装版代码的本地路径

【我现在最该做的】
等你告诉我代码路径，我立刻开始 RU-001。
```

## 注意事项

- 不要硬编码技能列表！每次都要真的扫一遍 `.persona/skills/` 目录
- 如果某个 skills/*.md 没有 frontmatter，跳过并在最后报告"X 个技能格式不规范"
- 只列【已激活】的技能（status 不是 ❌ 或 archived）
- 报告要简洁，不要把整个 PERSONA.md 复述一遍

## 这个文件的元用途

亲爱的 DogWind，你可能注意到了：这个文件比真正需要的【describe-myself】功能复杂一点。

那是因为它同时承担两个角色：
1. **真正可用的技能** — 你可以真的用它来自我介绍
2. **教学样本** — 让你看到一个"标准技能文件"长什么样

当你后面用 `001-skill-creator` 造新技能时，可以参考这个文件的结构作为模板。

但记住：**结构是骨架，不是教条**。如果某个技能不需要某一节，省略它。
