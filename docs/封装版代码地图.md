## 4️⃣ 输出格式？

### resultPicOut
- **8位灰度图**（CV_8UC1），大小 = modelInputW × modelInputH
- **不是热力图**，是**二值化 mask**：
  - 像素值 255 = 缺陷区域
  - 像素值 0 = 正常区域
- 阈值由 `halconRegionThreshMax` 控制，DLL 内部做：
```cpp
anomaly_map.masked_fill_(anomaly_map < halconRegionThreshMax, 0);
anomaly_map.masked_fill_(anomaly_map >= halconRegionThreshMax, 255);
```

### 函数返回值
- `1` = 有缺陷
- `0` = 无缺陷（此时 resultPicOut 全黑，调用方可跳过显示加速处理）

## 8️⃣ ⭐ 最可疑/最值得深挖的地方

### 🔴 最可疑 #1：`thresh = pow(thresh, 2.0)` — 阈值被平方了

infer 版 `anomalyDetMatin` 内部第一行：
```cpp
thresh = pow(thresh, 2.0);
```
用户传入 0.1，实际生效的是 0.01。**文档没提这个**，调参的人如果不知道，阈值行为会完全对不上。

### 🔴 最可疑 #2：输出是**灰度热力图**，不是二值 mask

infer 版的后处理流程：
```
clamp(0, 3.0) → /3.0 → GaussianBlur(sigma=2) → *255 → CV_8U → gammaCorrection(1.2)
```
输出是 0-255 的**连续灰度热力图**，不是 0/255 二值图。train 版的 `demo.cpp` 才做二值化。

这意味着：**调用方需要自己做 threshold 才能拿到缺陷 mask**（main 函数里确实做了 `cv::threshold(resultPicOut, resultPicOutth, 150, 255, THRESH_BINARY)`）。

### 🔴 最可疑 #3：封装版 4 参数接口隐藏了 5 个可调参数

导出函数：
```cpp
extern "C" int anomalyDetMatin(int modelIndex, cv::Mat& testImage, float anomalyThresh, cv::Mat& resultImage)
```
内部硬编码：
```cpp
const int modelInputW = testImage.cols;  // 用原图尺寸
const int modelInputH = testImage.rows;
int useCuda = 1;    // 强制 GPU
int overlap = 64;   // 固定重叠
int cutSize = -1;   // 自动计算
```
**被隐藏的参数就是推理速度的杠杆**——特别是 `modelInputW/H`（可以缩图加速）和 `cutSize`（影响分块数量）。

### � 最可疑 #4：`nearest_neighbors` 分块搜索 — infer 独有

infer 版新增了 `nearest_neighbors` 函数，将 Memory Bank 搜索拆成多次迭代：
```cpp
float spanlen1 = ceil(float(maxSearchNaxFeatrueSize1) / float(embedding_size));
int iterTimes = ceil(float(mb_size) / spanlen1);
```
`maxSearchNaxFeatrueSize1` 从 `modelConfig.yaml` 读取（默认 1.396e9），控制每次搜索的特征数量上限。
train 版直接 `torch::cdist` 一把搜完。**这是防 OOM 的关键优化，也是推理时间的隐藏瓶颈。**

### 🔴 最可疑 #5：`creatModel` 接收文件夹路径，不是文件路径

infer 版 `initializeModel` 自动拼接路径：
```cpp
modelPath = folderPath + "\\model.ckpt";
memorybankPath = folderPath + "\\model.mb";
configPath = folderPath + "\\modelConfig.yaml";
```
但 `接口说明.h` 写的是 `const char* path 模型路径`，容易被理解为文件路径。main 函数里甚至先传了父目录又传了子目录，说明开发者自己也不确定。

### 🟠 其他可疑点

1. **train/demo.cpp vs infer/demo.cpp 是两个不同版本**：train 版（1337行）是旧版，用 `torch::cdist` 直接搜索 + 二值化输出；infer 版（~600行 class）是新版，有 `nearest_neighbors` 分块搜索 + 热力图输出。**infer 版是实际部署的。**

2. **`maxDetectModel = 400`（infer）vs `4000`（train）**：train 版分配了 4000 个模型槽位，infer 版缩减到 400。

3. **Cython `.pyd` 和 `.py` 同时存在**：可能 `train.exe` 优先加载 `.pyd`（编译加速版），但源码 `.py` 也在，两者可能不同步。

4. **PaddleSeg 模型文件被故意改名**防信息泄露（`note` 文件明确说了）。

5. **`layerchoose` bug？**：`train.py` 第143行，`layerchoose[2]==1` 时追加 `'layer2'` 而不是 `'layer4'`，疑似复制粘贴错误。

6. **`gammaCorrection` 用 gamma=1.2**：这会让暗区更暗、亮区相对更亮，等于人为压低了低分缺陷的可见度。
