### 对比观察

1. **ret 完全一致**: Python 和 C++ 6 张全部 ret=1。确认这些都是异常样本。
2. **加速比 3.06-3.65x**: C++ 原版 (客户代码, 未优化) 已比 Python baseline 快 3.4 倍。
   - 大图 3.0x (受限于 resize/interpolate 开销)
   - 中图 3.6x (backbone forward 占比更高, C++ 优势更大)
3. **score 不可直比**: Python 用的是实际 anomaly score (1.09-1.17)，C++ harness 用的是 non-zero pixel ratio (1.0)。两者 metric 不同，但 ret 一致说明判断结果相同。

## 🔍 观察结论

### 好消息
- 6/6 ret 与 Python 对齐，**精度无偏离**
- C++ baseline 已有 3.4x 加速，证明 LibTorch + CUDA 路径工作正常
- 耗时非常稳定 (std < 3ms)

### 注意事项
- 6 张图仍然全部 ret=1 score=1.0。这不是 bug，Python 侧也全是 ret=1。
- 如需精度差异化测试，需要包含 **正常样本** (ret=0) 的测试图
- 客户 FP16 预处理 bug (L0' 诊断的 backbone max_diff=0.05) 对最终 ret 判断没影响——两条路径都给出 ret=1

### 性能瓶颈分布 (参考 Python 侧 RU-007 数据)
- Python: cdist 占 79% (233ms / 294ms)
- C++ 同理: KNN (nearest_neighbors 循环 cdist) 是主要瓶颈
- 预期 L3 cdist 优化后: 从 ~86ms 可降到 ~60ms 量级
