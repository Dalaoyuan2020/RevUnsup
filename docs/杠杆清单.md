## 总览

## 🔴 S 级杠杆（配置层，不改代码）

### S1：缩图推理（modelInputW/H）
- **位置**：封装版 `anomalyDetMatin` 硬编码 `modelInputW = testImage.cols`，即用原图尺寸
- **杠杆**：传入缩小的图 → 推理时间按面积比下降
- **风险**：缩太小会掉精度
- **操作**：调用方在送图前 resize，或修改 DLL 封装层
- **预估提速**：原图 2048×2048 缩到 1024×1024 → 理论 4× 提速

### S2：阈值调优（anomalyThresh）
- **位置**：`anomalyDetMatin` 参数，内部 `pow(thresh, 2.0)`
- **杠杆**：提高阈值 → 更多 patch 被 skip（`if amax1 < thresh: continue`），减少后处理
- **风险**：阈值太高会漏检
- **操作**：按阈值校准协议（Step E）找 F1 最优点
- **预估提速**：取决于缺陷率，正常图可快 2-5×

### S3：overlap 调整
- **位置**：封装版硬编码 `overlap = 64`
- **杠杆**：减小 overlap → 减少冗余计算区域
- **风险**：可能在 patch 边界漏检
- **操作**：修改 DLL 封装层或 modelConfig.yaml 中 `overlap` 字段
- **预估提速**：10-30%

### S4：coreset_sampling_ratio
- **位置**：`modelConfig.yaml` → `coreset_sampling_ratio: 0.1`
- **杠杆**：降低采样率 → Memory Bank 更小 → `nearest_neighbors` 更快
- **风险**：Memory Bank 太小会掉精度
- **操作**：重新训练时调整
- **预估提速**：线性关系，0.1→0.05 约 2× 搜索提速

## 🟠 A 级杠杆（需改代码）

### A1：FAISS 替换 torch::cdist
- **位置**：`nearest_neighbors` 函数中 `torch::cdist(embedding, memory_bank, 2)`
- **杠杆**：FAISS 的 IVF/PQ 索引比暴力搜索快 10-100×
- **风险**：近似搜索有精度损失
- **操作**：Python 端用 `faiss.IndexIVFFlat`，C++ 端用 faiss-gpu
- **预估提速**：10-50× 在 nearest_neighbors 步骤

### A2：Memory Bank 分块加载 → GPU
- **位置**：`MB2CPUORGPU` 函数，当前 Memory Bank 存 CPU（`mbType = 1`）
- **杠杆**：把 Memory Bank 直接放 GPU → 省去每次 `(*mbbeginiter).to(deviceGPU)` 的传输
- **风险**：显存占用增加
- **操作**：改 `mbType = 2` 或修改 `MB2CPUORGPU` 中的 `tmpMb` 直接 `.to(deviceGPU)`
- **预估提速**：减少每个 patch 的 CPU→GPU 传输，约 10-20%

### A3：backbone 换小模型
- **位置**：`con1.yaml` → `backbone: resnet18`，可选 `wide_resnet50_2`
- **杠杆**：resnet18 已经是小模型了，但可以试 MobileNetV3 / EfficientNet-B0
- **风险**：特征表达能力下降
- **操作**：修改 `con1.yaml` + `torch_model.py` 中 `FeatureExtractor`
- **预估提速**：2-3× 前向推理

### A4：去掉 gammaCorrection
- **位置**：`anomalyDetBigMatinPatchCoreCUT` 末尾 `gammaCorrection(anomaly, anomaly, 1.2)`
- **杠杆**：省一次全图遍历
- **风险**：输出热力图亮度分布变化，下游阈值需重新校准
- **操作**：注释掉这行
- **预估提速**：微小（1-3%），但简化了输出理解

### A5：num_neighbors 调整
- **位置**：`modelConfig.yaml` → `num_neighbors: 1`
- **杠杆**：已经是 1，无法再减。如果改成 >1 会更准但更慢
- **当前状态**：已经是最快配置，**此杠杆已用尽**

### A6：exportType 模式切换
- **位置**：`modelConfig.yaml` → `exportType: 1`（两段式：TorchScript + Memory Bank）
- **杠杆**：`exportType = 2` 是老版一段式 libtorch 推理，模型直出 anomaly_map
- **风险**：一段式模型可能更大，但省去 Memory Bank 搜索
- **操作**：需要重新训练导出
- **预估影响**：不确定，需实验

## 🟡 B 级杠杆（大改）

### B1：TensorRT 加速
- **位置**：当前用 LibTorch `module.forward()`
- **杠杆**：TorchScript → ONNX → TensorRT，FP16/INT8 量化
- **风险**：精度损失，部署复杂度增加
- **操作**：`torch.onnx.export` + `trtexec`
- **预估提速**：3-10× 前向推理

### B2：PaddleSeg TensorRT 加速
- **位置**：当前 `xyc_seg.dll` 用 Paddle Inference
- **杠杆**：Paddle Inference 本身支持 TensorRT 后端
- **操作**：`infer_config.EnableTensorRtEngine()`
- **预估提速**：2-5×

### B3：多模型并行推理
- **位置**：当前 `GlobleModels[maxDetectModel]` 支持多模型，但推理是串行的
- **杠杆**：如果有多张 GPU 或 CUDA Stream 并行
- **风险**：显存翻倍
- **操作**：改 DLL 架构

### B4：训练数据增强策略优化
- **位置**：`train.py` 中 `ColorJitter`, `RandomRotation`, `dataAugIters`
- **杠杆**：更好的数据增强 → 更小的 Memory Bank 也能达到同精度
- **风险**：需要大量实验
- **操作**：调整 `modelConfig.yaml` 中增强参数
