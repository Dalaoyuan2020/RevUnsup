# 阈值校准协议（强制规则）

> ⚠️ 这是规则，不是建议。任何涉及阈值的操作必须遵守。
> 任务：RU-003 | 日期：2026-04-09

---

## 为什么需要这个协议

`anomalyDet.dll` 内部有一个隐藏的阈值变换：

```cpp
// infer/demo.cpp → anomalyDetMatin()
thresh = pow(thresh, 2.0);
```

**传入值和实际生效值不一样。** 如果不显式追踪，调参就是盲打。

---

## Step A：显式声明 pow(thresh, 2) 陷阱

### 换算表

| 传入值 | 实际生效值 (传入²) | 备注 |
|--------|-------------------|------|
| 0.05 | 0.0025 | 极敏感 |
| 0.10 | 0.01 | main 函数默认值 |
| 0.15 | 0.0225 | |
| 0.20 | 0.04 | |
| 0.30 | 0.09 | |
| 0.50 | 0.25 | 较松 |
| 1.00 | 1.00 | 几乎不检出 |

### 强制标注规则

在任何文档、代码、commit message 中提到阈值时，必须写成：

```
thresh_input=0.1 (thresh_effective=0.01)
```

不允许只写一个数字。

---

## Step B：收集原始分数分布

### 目标
跑测试集，收集每个 patch 的 `nearest_neighbors` 输出的 `top_10.amax()` 值。

### 方法
1. 如果有旧版 `collect_scores.py`，复用
2. 如果没有，写一个 Python 脚本：
   - 加载 `model.ckpt`（TorchScript）和 `model.mb`
   - 对每张测试图跑前向 + nearest_neighbors
   - 记录每个 patch 的最大距离值
3. 输出 `docs/score_distribution.json`：
```json
{
  "scores": [0.003, 0.005, 0.12, 0.45, ...],
  "stats": {
    "min": 0.001,
    "max": 2.3,
    "mean": 0.05,
    "median": 0.02,
    "p95": 0.15,
    "p99": 0.8
  }
}
```

---

## Step C：提取当前阈值

从 `modelConfig.yaml` 读取所有阈值相关参数：

| 参数 | 含义 | 来源 |
|------|------|------|
| `choosedThreshFinal` | 最终置信度阈值 | modelConfig.yaml |
| `Thresh` | 嵌入阈值 | modelConfig.yaml |
| `Thresh1` | 第二嵌入阈值 | modelConfig.yaml |
| `anomalyThresh`（传入） | DLL 调用参数 | 调用方代码 |

注意：这些阈值作用在不同阶段，不要混淆。

---

## Step D：反平方算回真实传入值

已知：`实际值 = 传入值²`

反算：`传入值 = sqrt(实际值)`

### 示例

如果通过 Step E 得出最优实际阈值 = 0.04：
```
传入值 = sqrt(0.04) = 0.2
```

调用 DLL 时传 0.2。

---

## Step E：F1 最优阈值搜索

### 原则：反过度工程化

- ❌ 不做贝叶斯搜索
- ❌ 不做网格搜索
- ❌ 不做自动化超参优化
- ✅ 简单线性扫描，F1 max 就够

### 方法

1. 从 Step B 的分数分布中取范围 [min, p99]
2. 等间距取 20-50 个候选阈值点
3. 对每个候选阈值：
   - 阈值以上的 patch → 缺陷
   - 阈值以下的 patch → 正常
   - 与 ground truth 对比，算 precision / recall / F1
4. 取 F1 最大的阈值作为推荐值
5. 反平方算出传入值

### 输出

```json
{
  "recommended_thresh_input": 0.2,
  "recommended_thresh_effective": 0.04,
  "f1_at_recommended": 0.92,
  "precision_at_recommended": 0.95,
  "recall_at_recommended": 0.89,
  "scan_range": [0.001, 0.5],
  "scan_points": 50
}
```
