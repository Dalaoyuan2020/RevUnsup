## 八个待回答的问题

### 1️⃣ 时间线: 一个多星期里都做了什么?
按日期倒推, 列出你能识别的工作阶段。
格式:
```
2026-03-XX 到 2026-03-YY: <阶段主题>
 - 做了什么
 - 产出什么
```

如果没有明确的日期线索(commit/mtime/日志), 尽力推断, 标"推断"。

### 2️⃣ 实验清单: 具体跑了哪些实验?
任何能识别为"实验"的东西都列出来:
- 训练跑? 用了什么配置?
- 推理测试? 用了哪些样本?
- 参数扫描?
- A/B 对照?

每条实验要有: **名字 · 目的 · 是否跑完 · 关键数字(如果有)**

### 3️⃣ 有效发现: 哪些数字/结论值得记住?
- 精度数字
- 推理时间
- 超参的最佳值
- 数据集规模
- 任何羊爸爸在备注/代码里标注过"✓" "OK" "终于" 的地方

### 4️⃣ 失败记录: 哪些路不通?
**这一题和第 3 题同等重要, 甚至更重要。**
- 尝试过但跑不通的方法
- 看起来行但最后放弃的方向
- 被注释掉的代码里的意图
- TODO/FIXME 里留下的叹气
- commit message 里的情绪 ("再试一次" "终于不崩了" "放弃这条路" 等)

### 5️⃣ 可复用资产清单
能直接拿到 RevUnsup 主线用的东西:
- 脚本 (预处理/后处理/评估等)
- 数据集或数据子集
- 配置文件 (yaml/ini/json)
- 训练好的模型 checkpoint
- 日志中的样本分析

每条要写: **是什么 · 路径 · 多大 · 依赖 · 可直接用 / 需要适配**

### 6️⃣ 与 RU-001 封装版的对应
RU-001 发现了 3 个金矿 (见 docs/封装版代码地图.md):
- 金矿1: modelInputW/H 硬编码可绕过
- 金矿2: KNN 分块搜索是瓶颈 (maxSearchNaxFeatrueSize1)
- 金矿3: 阈值被平方 (pow(thresh, 2.0))

问题:
- 旧版实验里有没有观察到这些现象?
- 有没有人为踩过这些坑?
- 有没有数据能佐证或反驳这些发现?

如果你不知道 RU-001 是什么, 先去 github.com/Dalaoyuan2020/RevUnsup 读 `docs/封装版代码地图.md`。

### 7️⃣ ⭐ Top 3 复用
最该立刻拿到 RevUnsup 主线用的 3 件事。
**只选 3 件, 不要罗列更多。** 强迫自己做取舍。

格式:
```
复用 #1: <名字>
  · 是什么
  · 为什么现在就能用
  · 怎么拿过去
```

### 8️⃣ ⭐ Top 3 避坑
最该让 DogWind 和羊爸爸知道"不要再走这条路"的 3 件事。
**只选 3 件。**

格式:
```
避坑 #1: <描述>
  · 当时试了什么
  · 为什么不行
  · 如果重复会浪费多少时间
```
