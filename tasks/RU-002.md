## 三阶段设计

### Phase 1：准备

1. **同步数据集**：确认测试图片路径、数量、分辨率
2. **选数据集**：挑代表性样本（正常+缺陷各若干张）
3. **验证 .exe**：确认 `ADetect.exe` 或 DLL 能正常加载和运行
4. **确认模型路径**：找到已训练好的模型文件夹（含 `model.ckpt` + `model.mb` + `modelConfig.yaml`）

### Phase 2：Baseline 测量 → 锁定 baseline.json

1. 用封装版 DLL/EXE 跑全量测试集
2. 记录每张图的：
   - 推理时间（ms）
   - 返回值（0/1/-1）
   - resultPicOut 非零像素数
3. 输出 `docs/baseline.json`，格式：
```json
{
  "date": "2026-04-09",
  "model_path": "...",
  "thresh": 0.1,
  "images": [
    {"name": "xxx.jpg", "w": 2048, "h": 2048, "ret": 1, "time_ms": 320, "nonzero_px": 1234}
  ],
  "summary": {
    "avg_time_ms": 0,
    "max_time_ms": 0,
    "min_time_ms": 0,
    "defect_count": 0,
    "normal_count": 0
  }
}
```
4. **锁定 baseline**：此后所有优化的对比基准

### Phase 3：FAISS 接入（条件性）

- 前提：`src/faiss_inference.py` 已迁移完成
- 用 FAISS 替换 `nearest_neighbors` 的 `torch::cdist` 分块搜索
- 对比推理时间差异

## 三个陷阱（来自 RU-001）

### ⚠️ 陷阱 1：`thresh = pow(thresh, 2.0)`
用户传入 0.1，DLL 内部实际用 0.01。baseline 记录时**必须同时记录传入值和实际值**。

### ⚠️ 陷阱 2：输出是灰度热力图，不是二值 mask
DLL 输出 0-255 连续值，**不是** 0/255 二值图。如需统计缺陷面积，调用方需自己做 threshold。

### ⚠️ 陷阱 3：硬编码参数 baseline 阶段不能碰
- `useCuda = 1`
- `overlap = 64`
- `cutSize = -1`
- `modelInputW/H = 原图尺寸`

baseline 必须用原始封装版的默认行为，不改任何参数。
