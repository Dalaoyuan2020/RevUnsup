## 背景: 两套完全不同的阈值坑，同一个教训

### 坑 1: 封装版的 pow(thresh, 2.0) (来自 RU-001)
DogWind 在 `anomalyDetMatin` 函数第一行发现:
```cpp
thresh = pow(thresh, 2.0);
```
意味着：你传 0.1，函数实际用的是 0.01。文档没写。
所有不知道这个细节的调参实验，结论都是错的。

### 坑 2: 训练阈值 NaN (来自 OM-001)
OldMajor_Codex 在 `milestone1/thresholds.json` 看到:
```
raw_pixel_threshold = NaN
```
后来用 `collect_scores.py` 扫 acceptance 集重新校准，才得到可用的 19.1151。

### 共同教训
**训练完成 → 阈值可用** 这个假设是错的。
- 封装版坑你: 同一个数字含义被偷换 (平方)
- 旧版坑你: 数字本身是 NaN

两套坑都说: 部署前必须有【独立的阈值校准步骤】，不能信训练产物。

## 协议内容大纲 (建议结构)

### 1. 适用场景
触发协议的情况:
- 任何修改 `halconRegionThreshMax` / `Thresh` / `Thresh1` 的实验
- 任何更换测试集的实验
- 任何更换 .exe 版本的实验
- 任何修改归一化逻辑的实验

### 2. 输入物
- 一份带标注的【校准集】 (优先级: acceptance 集 > MVTec sanity 集 > selftest 集)
- 当前要校准的封装版 .exe / DLL
- 一份"原始阈值候选" (从训练产物或经验值)

### 3. 标准步骤

#### Step A: 平方陷阱声明
```
显式声明: 我知道封装版会做 thresh = pow(thresh, 2.0)
所以下面所有"传入阈值"都是【已平方前的值】
我要的"实际生效阈值" = 传入阈值的平方
```

#### Step B: 扫描分数
```
对每张校准集图:
  result = call_anomalyDet(image, thresh=0)  # 先用 thresh=0 拿原始分数
  record(image, label, raw_score)
```

#### Step C: 计算最优阈值
- 用 PR 曲线 / ROC 曲线
- 找出 F1 最大点 (或召回率约束下的最大精度点)
- 这个值就是【实际生效阈值】

#### Step D: 反向算传入阈值
```python
threshold_to_pass = sqrt(actual_threshold)  # 反平方
```

#### Step E: 校准结果文件
写到 `calibrations/<日期>_<exeVersion>_<datasetVersion>.json`:
```json
{
  "calibration_id": "cal-2026-04-09-v1",
  "exe_version": "...",
  "dataset_version": "...",
  "raw_score_distribution": {
    "good": {"min": ..., "max": ..., "mean": ..., "p95": ...},
    "bad":  {"min": ..., "max": ..., "mean": ..., "p95": ...}
  },
  "selected_threshold_actual": 0.0225,
  "selected_threshold_to_pass": 0.15,
  "rationale": "F1 max at this point",
  "metrics_at_this_threshold": {
    "precision": 0.85,
    "recall": 0.78,
    "f1": 0.81
  }
}
```

#### Step F: 写入主线
- 更新 `docs/baseline.json` 的阈值字段
- 在 Dashboard 编排端显示当前激活的校准 ID

### 4. 复用旧版的资产
旧版已经写好的脚本可以直接迁:
- `scripts/extract_thresholds.py` (从 checkpoint 抽阈值)
- `scripts/collect_scores.py` (扫 acceptance 拿分数)
- 旧版的 calibration_results.json 是【参考样本】

### 5. 反过度工程化
不需要做的事:
- ❌ 不需要做贝叶斯优化
- ❌ 不需要做多目标优化 (precision 和 recall 各一份)
- ❌ 不需要做 100 个候选阈值的网格搜索
- ✅ 只要 5-10 个候选 + F1 max 就够

### 6. 异常处理
- 如果分数全在一个区间 → 数据可分性问题，标注【不可校准】，不是【没校好】
- 如果出现 NaN → 立即报告，不要 fallback 到 0.5 (那是旧版踩过的坑)
- 如果校准集太小 (<10 张) → 报告"统计意义不足"

## ⚠️ 这份协议的真正意义

不是为了写一份"漂亮的协议文档"。
是为了让后续每一个跑 baseline / 调阈值 / 对比实验的人 (包括 Agent) 都【不会被同一个坑绊倒两次】。

旧版踩过 NaN 的坑。

RevUnsup 主线只能踩一次，剩下的都靠协议挡住。
