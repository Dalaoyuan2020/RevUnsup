## 背景

RevUnsup 项目从甲方拿到了两套代码：
1. **封装提交版**（已下载到本地）— 工业部署形态
2. **源码版**（已下载到本地，但有加密/白名单限制，详见 docs/L3-evaluation-spec.md §11.1）

后续所有优化动作（提精度+压推理时间）都需要先理解这两套代码的关系和接口。本任务负责【封装版】部分。

按 L2 编排的"问题理解"原则：在不知道当前长什么样之前，所有"下一步做什么"的决策都是盲打。

---

## 任务目标

输出一份《封装版代码地图》，作为后续所有任务的认知地基。

**产出物**：`docs/封装版代码地图.md`

**完成判定**：8 个问题都有答案（"不知道"也算答案，不要硬猜）。

---

## 八个待回答的问题

### 1️⃣ 入口在哪？
主程序文件名是什么？是 .exe / .bat / .py / GUI app / 服务进程 / 别的？

### 2️⃣ 怎么喂图进去？
- 命令行参数？
- 文件夹监听（hot folder）？
- GUI 拖拽？
- API 调用？
- 文件 watcher？

### 3️⃣ 输出在哪？
- 写到文件夹？
- 屏幕显示？
- 数据库？
- 网络发送？
- 日志文件？

### 4️⃣ 输出格式？
- JSON？
- 框坐标 (x,y,w,h)？
- mask 图（PNG/二值化）？
- 热力图？
- 分类标签？
- 置信度分数？
- 其他什么字段？

### 5️⃣ 依赖环境？
- .NET Framework / Core 版本？
- Python 版本？
- CUDA 版本？
- 显卡型号要求？
- ONNX Runtime / TensorRT 版本？
- 其他 DLL / SDK 依赖？

### 6️⃣ 配置文件在哪？
- 配置文件是什么格式（yaml/json/ini/xml）？
- 路径在哪？
- 哪些参数可调？(阈值/分辨率/batch/路径...)
- 哪些参数是 hardcode 在代码里的？

### 7️⃣ 跟"源码版"的对应关系？
- 封装版是源码编译/打包出来的吗？
- 还是两套独立的实现？
- 如果是编译出来的，编译入口在哪？
- 版本号是否一致？

### 8️⃣ ⭐ 你觉得【最可疑/最值得深挖】的地方？

**这一题最值钱。** 直觉性的观察，不需要证据。Agent 模仿不来博士生的直觉，但这正是 L2 编排"问题理解"能力的核心。

例子：
- 某个文件名特别奇怪
- 某个依赖看起来过时
- 某段流程明显冗余/可疑
- 某个参数像是临时硬编码
- 某个函数命名让人觉得有内幕
- 推理时间的瓶颈猜测在哪一步
